1. 안녕하세요. 멋쟁이사자처럼 백엔드 스쿨 1기에 지원한 박재현입니다. 현재 소속 및 상태에 대한 자기소개를 하겠습니다. 저는 졸업 후 1년간 SI 회사에서 RPA 개발을 맡았었습니다. 하지만 더 높은 성장 가능성을 목표로 퇴사를 결심하였고 현재는 백엔드 분야에 대해서 독학으로 공부중에 있습니다. 그러던 중에 백엔드 스쿨을 알게 되었고 프로젝트 경험이 부족한 저에게 좋은 과정이 될 것이라 생각해서 이렇게 지원하게 되었습니다. 특히 제가 백엔드 분야에 관심을 가지게 된 계기는 이전 회사에서 RPA로 프로세스를 개선한 경험이 있었는데 이때 많은 보람을 느꼈었습니다. 그래서 저는 이처럼 뒷단의 프로세스를 구현하고 개선하는 백엔드 분야에서 더욱 성장하고 싶다는 목표가 생기게 되었습니다.

2. 백엔드 스쿨을 수강하며 제가 진행하고 싶은 프로젝트는 쇼핑몰 프로젝트를 경험하고 싶습니다. 제 개인적인 생각으로 지난 2년간 디지털 전환이 가장 빠르고 효과적으로 일어난 분야는 이커머스 분야라고 생각합니다. 그렇기 때문에 앞으로도 성장 가능성이 높은 분야라고 생각합니다. 또한 결제, 상품 리스트, 회원 정보 등 다양한 필드를 경험해 볼 수 있을 것 같아 쇼핑몰 프로젝트를 진행하고 싶습니다. 이를 위해선 좋은 설계 기술이 필요할 것 같습니다. 물론 기본적인 프로그래밍 스킬도 중요하지만 좋은 설계가 있어야 결제, 상품 조회, 회원 검증 같은 프로세스가 꼬이지 않고 잘 만들어 질 수 있을 것 같습니다. 이처럼 프로젝트를 수행한다면 상품을 조회하는 부분과 결제 프로세스가 이루어지는 부분, 회원 정보를 업데이트하는 부분이 주요 개발 결과물이 될 것이라고 생각합니다.

3. 프로젝트를 진행하기 위한 제 학습 진행 방향은 조금이라도 배운 것을 활용해보는 것입니다. 이론 학습도 매우 중요하지만 저는 배운 것을 복습하고 작더라고 활용하는 경험이 있어야 몸에 더 빨리 익히고 이해가 더 빨리 된다고 생각합니다. 그렇기에 자바스크립트, 자바, 스프링의 한가지를 배우면 그것을 작게라도 활용해보며 몸에 익힐 수 있도록 학습을 진행하고자 합니다. 백엔드 스쿨의 교육과정에서도 자바 토이 프로젝트가 있는 것을 보았습니다. 이처럼 주입식 학습이 아닌 배운 것을 바로바로 활용해보는 교육과정이 좋은 방향이라고 생각합니다.

4. 제가 본 영상에서 설명할 문제는 7번 문제입니다. 모든 부분을 구현하고 싶었지만 아직 학습이 부족해서 게시판의 핵심 요소인 CRUD중 create 부분만 작성하였습니다. 3가지 API에 대해서 구현을 하였습니다. 먼저 홈화면의 경우 단순히 html 파일을 넘겨주게 됩니다. 해당 html에 글쓰기 포맷에 대한 타임리프로 앵커를 걸어서 앵커 클릭 시 글쓰기 폼을 전달해 달라는 GET 방식의 API가 요청되게 됩니다. 전달된 글쓰기 폼에서 사용자는 이름, 제목, 내용을 입력해서 html 폼을 POST 방식으로 전달하게 됩니다. 그러면 폼을 받아서 폼에 작성된 내용을 바탕으로 board라는 인스턴스를 만들어서 정보를 입력하게 됩니다. 컨트롤러는 이 보드 인스턴스를 서브시 계층에 전달하고 서비스 계층은 단순히 레포지토리의 인터페이스를 통해서 보드 인스턴스를 JDBC 레포지토리 구현 클래스로 이동시키게 됩니다. 그리고 mysql과 연결된 레포지토리 구현체에서 테이블에 insert하는 방식으로 코드를 작성하였습니다. 스프링 빈 등록의 경우 어노테이션 방식으로 등록하였고 레포지토리 계층에 인터페이스르 적용해서 서비스 계층에 자동으로 의존성 주입이 이루어 지도록 하였습니다.
